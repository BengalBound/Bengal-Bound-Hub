# Production Deployment Guide
# BengalBound HUB — BengalBound Ltd
**Version:** 1.1 | **Date:** June 2026 | **Owner:** CTO

---

## Overview

BengalBound uses a split deployment:

| Layer | Platform | Cost | URL pattern |
|---|---|---|---|
| Public marketing site | Cloud Run (free) | $0 | `bengalbound.com` |
| Django backend + all modules | Cloud Run (free → paid) | $0–$7/mo | `bengalbound-hub-*.run.app` |
| PostgreSQL database | Supabase (free) | $0 (500MB) | auto-injected |
| AI (33 agents + Serea) | Groq free tier | $0 | API calls |
| Static files | Whitenoise (served by Django) | $0 | `/static/` |
| DNS + CDN | Cloudflare (free) | $0 | — |

---

## Step 1 — Supabase (Database)

1. Sign up at **supabase.com** → New Project
2. Wait ~2 min for provisioning
3. Go to **Project Settings → Database → Connection String → URI**
4. Copy the `postgres://...` string — you'll paste it in Step 3

Free tier: 500MB storage, 2 shared CPUs, unlimited requests.

---

## Step 2 — Groq API Key (AI)

1. Sign up at **console.groq.com**
2. Go to **API Keys → Create API Key**
3. Copy the `gsk_...` key
4. Paste it in `.env` as `GROQ_API_KEY` locally
5. Add it to Cloud Run environment variables in Step 3

Free tier: 30,000 tokens/min on `llama-4-scout` — enough for all 33 agents.

---

## Step 3 — Cloud Run (Django Backend)

1. Sign up at **render.com**
2. New → **Web Service** → connect GitHub repo (`Adre-melech/BengalBound`)
3. Cloud Run auto-detects `render.yaml` — build and start commands are pre-configured
4. Set these **Environment Variables** in Cloud Run dashboard:

| Key | Value | How to get |
|---|---|---|
| `DATABASE_URL` | `postgres://...` | From Supabase Step 1 |
| `GROQ_API_KEY` | `gsk_...` | From Step 2 |
| `FIELD_ENCRYPTION_KEY` | run: `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` | Generate once, store safely |
| `SECRET_KEY` | Auto-generated by Cloud Run | Click "Generate" |
| `LITELLM_BASE_URL` | Leave blank for Groq direct mode | Optional |

5. Click **Deploy** — first build takes ~8 min (installs deps, migrates, seeds data)
6. Your app is live at `https://bengalbound-hub-*.run.app`

### Settings file used: `bengalbound_core/settings/render.py`
- Whitenoise for static files
- Celery tasks run eagerly (synchronous, no Redis needed)
- Email falls back to console log if `EMAIL_HOST` not set
- `AXES_ENABLED = False` (free tier has no persistent cache)

---

## Step 4 — Cloud Run (Public Site)

1. Connect custom domain in Cloud Run dashboard
2. New Site → **Import from Git** → connect GitHub repo
3. Cloud Run auto-detects `` — no config needed
4. Set one Environment Variable:
   - `FIELD_ENCRYPTION_KEY` — same key as in Cloud Run (must match)
5. Click **Deploy site**
6. Public site live at `https://your-site.com`

### What gets deployed to Cloud Run
The `export_static` management command renders all 12 public pages to static HTML:
`/`, `/about/`, `/services/`, `/pricing/`, `/contact/`, `/consult/`, `/hire-ai/`, `/affiliates/`, `/trial/`, `/why-us/`, `/docs/`, `/blog/`

---

## Step 5 — Custom Domain (Cloudflare)

1. Buy domain at **Cloudflare Registrar** (cheapest) or Namecheap
2. Add domain to Cloudflare free plan
3. In Cloudflare DNS:
   ```
   A      @        <cloud-run-ip>       (public site)
   CNAME  app      bengalbound-hub-*.run.app (backend)
   CNAME  www      ghs.googlehosted.com
   ```
4. In Cloud Run: Domain settings → Add custom domain → `bengalbound.com`
5. In Cloud Run: Settings → Custom Domain → `app.bengalbound.com`
6. SSL is automatic via Cloudflare + Let's Encrypt

---

## Step 6 — Email (Transactional)

Free option: **Brevo** (300 emails/day free)
1. Sign up at brevo.com → SMTP settings
2. Add to Cloud Run environment variables:
   ```
   EMAIL_HOST=smtp-relay.brevo.com
   EMAIL_HOST_USER=your@email.com
   EMAIL_HOST_PASSWORD=your-brevo-smtp-key
   EMAIL_PORT=587
   ```

---

## Step 7 — Stripe (Payments)

1. Sign up at **stripe.com** → Dashboard
2. Get keys from Developers → API Keys:
   ```
   STRIPE_PUBLISHABLE_KEY=pk_live_...
   STRIPE_SECRET_KEY=sk_live_...
   ```
3. Add to Cloud Run environment variables
4. Set up webhooks: Stripe Dashboard → Webhooks → Add endpoint → `https://app.bengalbound.com/stripe/webhook/`

For Bangladesh: set up **SSLCommerz** separately (see `docs/platform/PAYMENT_INTEGRATION.md`)

---

## Step 7b — Re-enable Email Verification

Email verification is currently **disabled** in `base.py` for development testing. Before going live, set it back:

```python
# bengalbound_core/settings/base.py
ACCOUNT_EMAIL_VERIFICATION = 'mandatory'   # change from 'optional' / 'none'
```

Also verify that `EMAIL_HOST` and `EMAIL_HOST_USER` are set in the production environment (Step 6 above) so verification emails actually send.

Run these after first deploy:

```bash
curl -s -o /dev/null -w "%{http_code}" https://app.bengalbound.com/

```

## Scaling Up (When Free Tier Isn't Enough)

| When | Action | Cost |
|---|---|---|
| > 100 concurrent users | Upgrade Cloud Run to Starter ($7/mo) | $7/mo |
| > 500MB database | Upgrade Supabase to Pro ($25/mo) | $25/mo |
| > 30k tokens/min AI | Upgrade Groq to Dev tier | ~$5/mo per 1M tokens |
| Heavy AI usage | Add self-hosted Ollama on Hetzner VPS | $19–56/mo fixed |
| > 1000 clients | Cloud Run → Dedicated + PostgreSQL replica | $50–200/mo |

See `docs/business/VPS_PROFIT_MODEL.md` for detailed cost projections.

## Environment Variables Reference

| Variable | Required | Dev default | Production |
|---|---|---|---|
| `SECRET_KEY` | Yes | Set in `.env` | Cloud Run auto-generates |
| `DEBUG` | No | `True` | `False` (auto in render.py) |
| `DATABASE_URL` | Prod only | blank (uses SQLite) | Supabase postgres:// |
| `GROQ_API_KEY` | For AI | Set in `.env` | Set in Cloud Run |
| `FIELD_ENCRYPTION_KEY` | Yes | Set in `.env` | Set in Cloud Run |
| `LITELLM_BASE_URL` | No | blank (uses Groq direct) | Optional proxy URL |
| `LITELLM_MASTER_KEY` | No | blank | Optional proxy key |
| `EMAIL_HOST` | Prod only | blank (console) | Brevo SMTP |
| `ALLOWED_HOSTS` | Prod only | blank | `bengalbound.com,app.bengalbound.com` |
| `STRIPE_SECRET_KEY` | For payments | blank | Stripe live key |
