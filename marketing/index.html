// How It Works

Hire once. Works forever.

Every AI employee is auto-provisioned the moment you subscribe. No setup. No configuration. They start working immediately.

01
Choose your modules and agents
Activate the business modules your team uses and hire the AI agents that match your roles. Each agent knows which module data to work with.
02
Agents start working autonomously
Every agent runs on its own Celery schedule — checking for tickets, scoring leads, reviewing contracts, monitoring pipelines — 24 hours a day.
03
Approve sensitive decisions
When an agent is uncertain, it pauses and emails you for approval. You click Approve or Deny. Full audit trail kept for every decision.
// Integrations

Deploy agents anywhere.

Every agent exposes a REST API and accepts inbound webhooks. Use them inside BengalBound HUB or connect them to any external system.

🧩
Inside BengalBound Modules
Agents read and write to module data directly via Django ORM — no API overhead, real-time sync. Crux scores your CRM contacts. Merch optimises your product listings.
agent → modules.crm.Contact
agent → modules.inventory.Product
🔌
REST API — Any External System
Every agent exposes DRF endpoints. Send a ticket, get a resolution. Send a prospect, get a scored outreach sequence. Works with any language or framework.
POST /hub/acme-corp/agents/aria/tickets/
Authorization: Token <your-token>
🪝
Inbound Webhooks — Event-Driven
Register a webhook endpoint for any agent. Point Shopify, GitHub, Stripe, DocuSign, or Recall.ai at it. The agent handles the event immediately.
POST /agents/webhook/<token>/
X-Hub-Signature-256: sha256=...
⚙️
Standalone — No Module Required
Set an agent instance to idle and wire in encrypted external credentials. It runs on its own Celery Beat schedule, pulling data from wherever you configure.
AgentIntegration(platform='shopify',
credential=encrypted_token)
